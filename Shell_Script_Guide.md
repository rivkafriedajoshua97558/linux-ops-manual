## 1. Shell 核心前置认知
### 1.1 理解 Shell、内核与终端的关系
在日常运维中，我们经常接触这三个名词，理解它们的关系是学号 Shell 的第一步：
- **系统内核 (Kernel)**：类比智能写字楼的“底层基础设施”（水电、安保、电梯），是系统的核心，用户无法直接操作。
- **Shell (解释器)**：类比写字楼“前台客服”，负责接收用户命令、翻译指令并传递给内核执行，最后返回执行结果。
- **终端 (Terminal)**：前台交互的窗口，是用户输入命令和查看输出结果的可视化界面。
- **Shell 脚本 (Script)**：预先编写好的批量任务清单，可实现自动化批量执行。

### 1.2 核心认知落地
> **官方定义**：Shell 是 Linux 系统的命令行解释器，作为用户与系统内核之间的交互桥梁。

## 3. 第一个 Shell 脚本与基础操作
### 3.1 脚本规范首行 (Shebang Line)
    ```bash
    #!/bin/bash
    ```
> 作用：告诉系统，当前脚本必须交由 /bin/bash 这个 Shell 解释器来执行。

### 3.2 脚本的三种运行方式

1. **方式一：授权后相对路径执行（企业推荐）**
    ```bash
    chmod +x hello.sh
    ./hello.sh
    ```
2. **方式二：解释器直接执行（无需授权）**
    ```bash
    bash hello.sh
    sh hello.sh
    ```
3. **方式三：source/点执行（加载到当前终端）**
    ```bash
    source hello.sh
    . hello.sh
    ```
> 💡 核心区别：前两种方式会**创建子 Shell 执行**，而第三种方式在**当前 Shell 中执行**，会影响当前环境变量。

### 3.3 数据流重定向与管道操作
- **覆盖重定向 (>)**：清空文件原有内容，写入新内容。
- **追加重定向 (>>)**：在文件末尾追加新内容（常用于日志写入）。
- **错误重定向 (2>)**：单独收集命令报错信息。
- **全量重定向 (&>)**：统一收集正确和错误输出。
- **管道符 (|)**：将前一个命令的输出，直接作为后一个命令的输入。
    ```bash
    # 示例1：筛选包含 bash 的进程
    ps -ef | grep bash

    # 示例2：统计当前目录下脚本文件的数量
    ls | grep ".sh" | wc -l
    ```

## 4. Shell 变量核心与高级扩展
### 4.1 变量定义规范
> **关键规则**：变量名=变量值，等号**两侧绝对不能有空格**。
> **取值**：使用 $变量名 或 ${变量名}（推荐，避免歧义）。

### 4.2 三大变量类型
1. **自定义本地变量**：仅在当前脚本或终端会话生效，作用域最小。
2. **环境变量**：全局生效，所有子进程和脚本均可调用，使用 export 声明。例如 export PATH=$PATH:/new/bin。
3. **位置参数变量**：脚本执行时通过命令行传入的参数（用 $1、$2 等获取）。例如 ./script.sh 参数1 参数2。

### 4.3 高级变量扩展（核心利器）
1. **变量默认值兜底**：使用 :- 默认值语法，防止变量为空导致报错。
    ```bash
    MSG=""
    echo "提示：${MSG:-暂无数据}"  # 输出：提示：暂无数据
    ```
2. **变量截取（文件名/字符串处理）**
    ```bash
    FILE="/home/demo.tar.gz"
    echo ${FILE##*/}  # 截取文件名 demo.tar.gz
    echo ${FILE%.*}   # 截取后缀前的部分 /home/demo.tar
    ```
3. **命令结果动态赋值**：使用 $() 命令替换。
    ```bash
    TIME=$(date "+%Y-%m-%d")
    echo "当前时间：${TIME}"
    ```

## 5. Shell 运算符与测试判断
### 5.1 算术、关系与字符串运算符
- **算术运算**：推荐使用 $(( )) 语法进行整数运算（加、减、乘、除、取余）。
    ```bash
    A=20; B=5
    echo "加法：$((A+B))"  # 输出25
    ```
- **关系运算**：用于数值比较（-eq 等于，-gt 大于，-lt 小于）。推荐统一使用 [[ ]] 现代语法。
    ```bash
    [[ 10 -gt 5 ]] && echo "Yes"  # 输出 Yes
    ```
- **字符串运算**：用于判断字符串是否相等，或判断是否为空（-z, -n）。
    ```bash
    STR1="Shell"; STR2="Shell"
    [[ $STR1 == $STR2 ]] && echo "匹配成功"
    ```
### 5.2 文件测试与判断演进（运维高频）

- **文件测试运算符**：-d (判断是否为目录)、-f (判断是否为文件)、-e (判断文件或目录是否存在)。
- **判断语法三代演进**：
  1. test 原始语法（书写繁琐，易出错）。
  2. [ ] 单中括号（兼容旧语法）。
  3. [[ ]] 双中括号（**现代推荐**）：功能强大，支持正则匹配、多条件拼接，且避免了变量为空引发的语法错误。

## 6. 流程控制 if 判断语句

> 核心：通过条件真（True）或假（False）决定程序的执行流向。

    ```bash
    # 1. 单分支 if
    if [[ $USER == "root" ]]; then
        echo "当前为超级管理员"
    fi

    # 2. 双分支 if-else
    if [[ -f $FILE ]]; then
        echo "文件已存在"
    else
        touch $FILE; echo "已创建文件"
    fi

    # 3. 多分支 if-elif-else
    if [[ $SS -ge 90 ]]; then
        echo "优秀"
    elif [[ $SS -ge 60 ]]; then
        echo "及格"
    else
        echo "不及格"
    fi
    ```

## 7. 循环语句 for 与 while
### 7.1 for 循环：遍历与批处理
用于已知集合或数字区间的遍历，非常适合批量创建文件、修改文件名、批量重启服务。
    ```bash
    # 基础遍历（字符串）
    for i in item1 item2 item3; do
        echo $i
    done

    # 数字区间遍历
    for i in {1..5}; do
        touch "test_${i}.txt"
    done
    ```
### 7.2 while 循环：基于条件的持续执行

适用于条件成立时就不断执行，或者处理未知循环次数的场景。

> 🚨 注意：必须有条件变更逻辑（如自增），避免死循环。

    ```bash
    # 示例：循环输出 1~5
    NUM=1
    while [[ $NUM -le 5 ]]; do
        echo "当前数字：$NUM"
        let NUM++  # 变量自增
    done
    ```
### 7.3 循环控制命令
- break：立即退出并终止整个循环结构。
- continue：跳过当前循环的剩余代码，直接开始下一次循环条件判断。

## 8. Shell 函数 代码复用核心
将重复使用的代码封装成函数，实现一次定义、多次调用，大幅提高脚本可读性与可维护性。
    ```bash
    # 1. 基础定义与无参调用
    SHOW_MSG(){ echo "调用成功!"; }
    SHOW_MSG

    # 2. 灵活带参调用
    CALC(){ echo "和为：$(( $1 + $2 ))"; }
    CALC 10 20  # 输出：和为 30
    ```
> 💡 核心：函数内部通过 $1、$2 等位置变量接收外部传入的参数，让函数更通用。

## 9. Shell 三剑客入门 (grep/sed/awk)

运维中的日志分析、文件修改和数值提取，全靠三剑客。
(详细用法请参考项目中 Linux_Ops_Manual.md 中的附录 6、7、8)

- **grep：文本过滤筛选（查找）**
    擅长找内容，快速筛选出包含关键字的行。常用参数：-v (反向), -i (忽略大小写), -c (统计行数), -E (支持多正则匹配)。
    ```bash
    grep -i -n 'error' /var/log/nginx/error.log
    ```
- **sed：流编辑器（修改）**
    擅长改内容，批量对文本进行增、删、改、替换。必备参数 -i.bak 进行原地带备份修改。
    ```bash
    sed -i.bak 's/80;/8080;/g' /etc/nginx/nginx.conf
    ```
- **awk：文本格式化截取（计算）**
    擅长取数据、做统计，按列拆分、精准截取指定字段，并能直接做数学运算（算 CPU/内存百分比）。
    ```bash
    free -m | awk '/Mem:/{print $2}'  # 取出总内存
    ```
- **组合实战**：grep筛选 → sed清洗 → awk提取，结合管道 | 可轻松完成复杂的日志与数据自动化分析。

## 10. 核心编码规范与避坑指南
> 规范是“防错”的基础，调试是“排错”的关键。
### 10.1 基础编码规范（杜绝 80% 新手报错）
- **Shell 脚本首行**：必须写 #!/bin/bash。
- 空格规范：[[、]] 内部、赋值等号两侧、条件运算符两侧都必须留一个空格。
- 严谨引用变量：强烈建议变量必须加双引号 "$VAR"，并使用花括号 ${VAR} 防止变量名混淆。
- 多写注释：关键逻辑加 # 注释，提升代码可读性。

### 10.2 调试排错技巧（透视眼）
- 开启调试模式：执行命令前加 bash -x script.sh，或者在脚本顶部加 set -x，**会实时打印每一条执行的命令**，对排查逻辑错误极有帮助。
- 分步执行：遇到复杂的逻辑块，先验证单个命令再写入脚本，定位错误。不忽略报错信息，聚焦于多行提示。

### 10.3 新手避坑指南与实战落地的核心技巧
1. **文件操作先判断**：创建目录、删除文件前，使用 -d 或 -f 判断，避免路径不存在导致执行中断。
2. **杜绝 Windows 脚本直接上传**：Windows 换行符为 \r\n，Linux 为 \n。必须用 dos2unix 转换后再运行，否则控制台会报无权限或奇怪的错误。
3. **路径优先绝对路径**：避免变量失效或相对路径改变导致的执行错位，确保文件操作的绝对准确性。
4. **空变量要加默认值**：使用 ${VAR:-默认值} 语法，防止变量为空时引发逻辑错误或 -z 判断失灵。
5. **重要操作加交互确认**：关键步骤添加 read -p 确认逻辑，防止误删、误改带来的不可逆损失。
    ```bash
    read -p "确定要停止 Nginx 服务吗？(y/n) " confirm
    [[ $confirm == "y" ]] && systemctl stop nginx
    ```
