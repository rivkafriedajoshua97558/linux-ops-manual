## 1. grep：文本过滤筛选（查找）

> **核心定位**：擅长从大量杂乱的日志、文件或命令输出中，快速筛选出包含指定关键字的行。

### 1.1 核心语法
```bash
grep [选项] '搜索关键字' 文件名
命令 | grep [选项] '搜索关键字'
```
### 1.2 最常用的 6 个参数
| 参数	| 作用                                                                            | 实战示例
| :---:         | :---:                                                                            |:---:
| `-v`	| **反向匹配**（排除法），输出不包含关键字的行             | `grep -v '^#' /etc/nginx/nginx.conf `(过滤掉注释)
| `-i`	| **忽略大小写**                                                             | `grep -i 'error' /var/log/nginx/error.log`
| `-n`	| **显示匹配行的行号**（便于快速定位编辑）                   | `grep -n 'root' /etc/nginx/nginx.conf`
| `-c`	| **只统计匹配的行数**（替代 wc -l，效率更高）              | `grep -c '192.168.1.101' access.log`
| `-r`	| **递归搜索**（整个目录）	                                    | `grep -r 'listen 8080' /etc/nginx/`
| `-C n`	| **显示上下文**（n为行数），查错时看前后内容              | `grep -C 3 ' 404 ' /var/log/nginx/access.log`
### 1.3 正则匹配实战

-     **精准匹配**：前后加空格，避免匹配到文件名或时间戳。`grep " 404 " access.log`

-     **多关键字匹配**：使用 `-E` 或 `egrep`，用 `|` 表示“或”。`grep -E '192.168.1.101|192.168.1.102' access.log`

-     **行首与行尾**：`grep '^Error' error.log` (匹配以 Error 开头的行)；`grep 'end$' error.log` (匹配以 end 结尾的行)。

### 1.4 进阶运维骚操作

-     **过滤掉自身**：`ps aux | grep nginx | grep -v grep` (或者更优雅的：`ps aux | grep '[n]ginx'`)

-     **搜索压缩包日志**：因为没解压无法用 `grep`，必须用专属命令：`zgrep ' 404 ' /var/log/nginx/access.log.20251222.gz`

## 2. sed：流编辑器（修改）

>    **核心定位**：擅长无需打开文件，直接在命令行中**批量增、删、改、替换**文件内容。是 Shell 脚本自动化修改配置的核心。

### 2.1 核心语法与格式
```bash

sed [选项] '动作' 文件名
```
*⚠️ 重中之重：`-i` 表示“原地修改”（直接改文件）。必备绝招是 `-i.bak`，修改前先备份成` .bak` 文件，防止写错正则导致文件被清空！*
### 2.2 运维高频动作

1. **替换（使用率 90%）**
`sed 's/查找内容/替换内容/标志'`

-     **基础替换**：`echo "hello world" | sed 's/world/linux/'`

-     **全局替换（加 g 标志）**：`echo "apple apple" | sed 's/apple/orange/g'`

🚀 **实战场景：一键更换 NTP 服务器 (完美契合 `chrony.conf`)**
```bash

# 将原文件中以 server 开头的行，全部替换为阿里云 NTP，并生成 .bak 备份
sed -i.bak 's/^server.*/server ntp.aliyun.com iburst/g' /etc/chrony.conf
```
2. **删除行**
sed '行号/正则/d'

-     删除第 2 行：sed '2d' file.txt
  
-     删除包含 error 的行：sed '/error/d' file.txt

1. **插入与追加**

-     第 2 行后面追加：sed '2a\新内容' file.txt

-     第 2 行前面插入：sed '2i\新内容' file.txt

4. **多命令协同 (-e)**
```bash
# 替换端口为 2222，同时删除所有以 # 开头的注释行
sed -i.bak -e 's/Port 22/Port 2222/g' -e '/^#/d' /etc/ssh/sshd_config
```

## 3. awk：文本格式化截取与计算（提取）

>    **核心定位**：按列切分文本，提取特定列的数据，并且具备直接做数学运算的能力。在脚本中计算 CPU 和内存百分比时必不可少。

### 3.1 核心语法与常用内置变量
```bash
awk '条件 { 动作 }' 文件名
命令 | awk '条件 { 动作 }'
```

运维必懂的 4 个内置变量：
-     $0：代表整行内容。
-     $1, $2... ：代表按空格切分后的第1列、第2列。
-     $NF：代表**最后一列** (Number of Fields)。
-     NR：代表当前是**第几行** (Number of Records)。

### 3.2 运维实战场景

🚀 **场景1：从 free -m 提取内存数据**
```bash
MEM_TOTAL=$(free -m | awk '/Mem:/{print $2}')
MEM_USED=$(free -m | awk '/Mem:/{print $3}')
```
🚀 **场景2：动态抓取 CPU 空闲率（无视列号变化，极稳）**
不要死记 $8，让 awk 自动去找 id, 这个标识符所在的前一列：
```bash

CPU_IDLE=$(top -b -n1 | awk '/%Cpu/ {for(i=1;i<=NF;i++) if($i=="id,") print $(i-1)}')
```
🚀 **场景3：计算百分比并保留整数（运维绝杀技）**
结合 BEGIN 块，在 awk 内部直接完成数学运算：
```bash
MEM_USAGE=$(awk "BEGIN {printf \"%.0f\", ($MEM_USED / $MEM_TOTAL) * 100}")
# printf "%.0f" 表示保留 0 位小数，即四舍五入为整数
```
🚀 **场景4：提取文件特定列**
处理 /etc/passwd 时，因用冒号 : 分隔，需使用 -F 指定分隔符：
```bash
awk -F':' '{print $1, $6}' /etc/passwd
```

## 4. 三剑客对比总结与组合实战
### 4.1 三剑客对比表（面试高频题）
| 对比维度	              | grep (查找)	       | sed (修改)	                | awk (提取与计算)
| :---:        |:---:     |:---:                                |:---:
| 核心职责	              | 过滤：找出包含关键字的行 | 编辑：增删改查配置          | 提取：按列切割并计算
| 操作单位	              | 按 行 过滤                         | 按 行 修改                        | 按 列 提取数据
| 是否支持数学运算   | 不支持	                       | 弱                                    | 极强 (printf, 加减乘除)
| 是否修改原文件      | 不能修改	                       | 能 (配合 -i)                        | 不能修改（仅做分析）
| 最佳运用场景         | 找报错行、查 IP 记录	       | 批量替换端口、去掉 % 符号| 从 top、free 抓数值算百分比
### 4.2 三剑客组合实战（排错流水线）

在实际运维中，极少用一个命令解决所有问题，它们经常通过管道符 | 串联使用，形成强大的数据**清洗流水线**：

需求：在 Nginx 的访问日志中，找出访问次数最多的前 5 个 IP 地址，并统计每个 IP 访问了多少次。
```bash

# 1. grep 过滤（排除掉健康检查的无关 IP）-> 2. awk 提取第 1 列 IP -> 3. sort 排序 -> 4. uniq -c 统计次数 -> 5. sort -nr 按次数降序 -> 6. head -5 取前 5 名
grep -v "127.0.0.1" /var/log/nginx/access.log | awk '{print $1}' | sort | uniq -c | sort -nr | head -5
```
总结：`grep` 搞定筛选，`sed` 搞定配置修改，`awk` 搞定业务数值的转换与统计。
